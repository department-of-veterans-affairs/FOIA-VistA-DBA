Subject: INTERIM ADDRESS ENHANCEMENTS FOLLOW UP

Category: 
  - Routine

Description:
============

 This patch is a follow up to the Interim Address Enhancement project.
  
 FUNCTIONAL DESCRIPTION
 ======================
 The functional changes introduced with this patch include:
   
 1. A count that indicates the total number of records listed in the 
 report will be added at the end of the IVM Address Updates Pending Review 
 [IVM ADDR UPDT PENDING REVIEW] report.
  
 2. A new mail group IVM ADDR UPDT REPORT will be added. The patch will not
 add any local or remote member to the mail group but members must be 
 added prior to generating the report.
  
 If the mail group has no member, the report will not be generated:
   Select OPTION NAME: IVM ADDR UPDT PENDING REVIEW
   IVM ADDR UPDT REPORT does not have a member. Report not sent.
  
 3. The IVM Address Updates Pending Review [IVM ADDR UPDT PENDING REVIEW] 
 report will be delivered to the IVM ADDR UPDT REPORT mail group, whether 
 the report is run through TaskMan or via the IVM Address Updates 
 Pending Review [IVM ADDR UPDT PENDING REVIEW] option.
  
 4. If run via option IVM Address Updates Pending Review [IVM ADDR UPDT 
 PENDING REVIEW], the task will be queued to be sent to the IVM ADDR UPDT 
 REPORT mail group. The user will be prompted to select a requested start 
 time.

 INSTALLATION INSTRUCTIONS 
   
          * * * * * *  A  T  T  E  N  T  I  O  N  * * * * * * 
    
 It is recommended this patch be installed outside of normal business 
 hours to avoid any complications resulting from users on the system. 
 Installation will take approximately 5 minutes.
   
 This patch should NOT be installed while the IVM BACKGROUND JOB [IVM 
 BACKGROUND JOB] is running.
   
 In order to prevent missing HL7 traffic while this patch is being 
 installed, TaskMan should be placed in a wait state.
  
 Please make sure that the one time clean up job introduced by IVM*2.0*79
 is not running when the patch is being installed. The one time clean up 
 job of IVM*2.0*79 is default to be run 30 days after IVM*2.0*79 is 
 installed, yet the task can be re-queued.
   
  Installation Steps
  ==================
  1.  Use the INSTALL/CHECK MESSAGE option on the PackMan menu. [Note: 
  TEXT PRINT/DISPLAY option in the PackMan menu will display the patch text
  only.] 
   
  2. Review your mapped set.  If any of the routines listed in the ROUTINE 
  SUMMARY section of patches IVM*2.0*93, they should be removed from the 
  mapped set at this time.
   
  3. From the 'Kernel Installation & Distribution System' menu, select the 
  Installation menu.
   
  4. From this menu, you may elect to use the following options (when 
  prompted for INSTALL NAME, enter IVM*2.0*93).
   
       a. Backup a Transport Global - This option will create a backup
          message of any routines exported with the patch.  It will NOT
          backup any other changes such as DDs or templates.
   
       b. Compare Transport Global to Current System - This option will
          allow you to view all changes that will be made when the patch
          is installed.  It compares all components of the patch
          (routines, DDs, templates, etc.).
   
       c. Verify Checksums in Transport Global - This option will allow
          you to ensure the integrity of the routines that are in the
          transport global.
   
       d. Print Transport Global - This option will allow you to view the
          components of the KIDS build.
   
  5. Use the Install Package(s) option and select IVM*2.0*93.
   
  6. When prompted 'Enter the Coordinator for Mail Group 'IVM ADDR UPDT 
  REPORT':', please enter the name of the coordinator. 
   
  7. When prompted 'Want KIDS to INHIBIT Logon's during the install? 
  YES//', it is recommended you answer NO.
   
  8. When prompted 'Want to DISABLE Scheduled Options, Menu Options, and 
  Protocols? YES//', respond YES.  When prompted to select the options you 
  would like to place out of order, enter the following:
   
          IVM BACKGROUND JOB
   
  9. If routines were unmapped as part of Step 3, they should be returned 
  to the mapped set once the installation has run to completion.
   
  10. Return TaskMan to it's normal state.
   
  11. If you unscheduled the IVM Background Job, please reschedule it. 
  
 POST-INSTALLATION INFORMATION
 ==============================
  The patch will create a local mail group IVM ADDR UPDT REPORT, without
  adding any members to the mail group. Please add members to the mail
  group before using the mail group. To add members to the mail group:
   
   Select Group/Distribution Management Option: MAIL GROUP EDIT
   Select MAIL GROUP NAME: IVM ADDR UPDT REPORT
   MAIL GROUP NAME: IVM ADDR UPDT REPORT Replace
   Select MEMBER: (please add in all the members to whom the IVM Address 
   Updates Pending Review [IVM ADDR UPDT PENDING REVIEW] report should be 
   sent)

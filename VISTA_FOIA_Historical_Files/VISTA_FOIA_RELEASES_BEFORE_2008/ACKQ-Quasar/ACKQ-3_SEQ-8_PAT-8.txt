Description:
============

 This patch is part of the Code Text Descriptors (CTD) Project. This patch 
 is NOT part of a build and should be installed independently.
 Please review the Installation heading of this patch for Installation 
 Guide retrieval information for the CTD software.
  
  
 This patch contains the following enhancements and problem resolutions:
  
 1) Only allow the display of the correct version of ICD9 and CPT 
 Modifiers code descriptions 
 =================================================================
  
 New Clinic Visits [ACKQAS VISIT ENTRY]
 Edit an Existing Visit [ACKQAS VISIT EDIT]
  
 The new Code Text Descriptor API's are now called when displaying CPT 
 Modifier's and ICD9 code text within the New Clinic Visits [ACKQAS VISIT
 ENTRY] and the Edit an Existing Visit [ACKQAS VISIT EDIT] options.  The
 date used when calling the Code Text Descriptor API's is the Visit Date of
 the Quasar encounter.   Within these options ICD9 code text is displayed
 when entering ICD9 codes that are new to the patient, editing previously
 entered ICD9 codes, editing previously entered CPT modifier codes and
 within the patient inquire section that is displayed at the beginning of
 both options.
  
 The following routines have been updated:
  
 ACKQUTL4 - This routine has been enhanced to call ACKQUTL8.
 ACKQASU  - This routine has been enhanced to call ACKQUTL8.
 ACKQNQ   - This routine has been enhanced to call ACKQUTL8.
 ACKQUTL7 - This routine has been enhanced to call ACKQUTL8.
 ACKQUTL8 - This routine has been enhanced to call the Code Text Versioning
 API's.
  
 2) Display the correct version of ICD9 code description 
 ======================================================= 
  
 Inquire - A&SP Patient [ACKQAS PAT INQ]
  
 The ICD-9 code text displayed during the Inquire - A&SP Patient [ACKQAS 
 PAT INQ] option shall be valid for the operation based on the date the 
 ICD-9 code was entered into the patients Quasar diagnostic history.
  
 The following routines have been updated:
  
 ACKQNQ   - This routine has been enhanced to call ACKQUTL8.
 ACKQUTL7 - This routine has been enhanced to call ACKQUTL8.
  
  
 3) Send the correct version of the ICD9 code description
 ========================================================
  
 New Clinic Visits [ACKQAS VISIT ENTRY]
 Edit an Existing Visit [ACKQAS VISIT EDIT]
  
 When Quasar encounter data is sent to PCE the correct version of the ICD9
 code(s) long description (based on the date of the encounter) is sent as
 the narrative field.
  
 The following routine has been updated:
  
 ACKQPCE1 - This routine has been enhanced to call ACKQUTL8.
 ACKQUTL8 - This routine has been enhanced to call the Code Text 
 Versioning  API's.
  
 4) Display the correct version of ICD9 code description
 =======================================================
  
 Visits by Diagnosis  [ACKQAS VISITS BY DIAG]
  
 The ICD-9 code text displayed  during the Visits by Diagnosis  [ACKQAS 
 VISITS BY DIAG] option will be valid for the operation based on the date 
 that the report is run.
  
 ACKQR3 - This routine has been enhanced to call ACKQUTL8.
  
  
 5) Display the correct version of the CPT code description
 ==========================================================
  
 Cost Comparison Report [ACKQAS PRINT COST COMPARE]
  
 The CPT code text displayed within the Cost Comparison Report [ACKQAS 
 PRINT COST COMPARE] option will be valid for the operation based on the 
 date that the report is run. 
  
 ACKQR4 - This routine has been enhanced to call ACKQUTL8.
  
 6) Display the correct version of the CPT code description
 ==========================================================
  
 Statistics by Procedure [ACKQAS PROC STATS]
  
 The CPT code text displayed within the Statistics by Procedure [ACKQAS 
 PROC STATS] option will be valid for the operation based on the date that 
 the report is run. 
  
 ACKQR2 - This routine has been enhanced to call ACKQUTL8.
  
 7) Display the correct version of the CPT Modifier code description
 ===================================================================
  
 Update CPT Modifiers per CO Directive [ACKQAS UPDATE CPT MODIFIERS]
  
 The CPT Modifier code text displayed within the Update CPT Modifiers per
 CO Directive [ACKQAS UPDATE CPT MODIFIERS] option will be valid for the
 operation based on the current date. 
  
 [ACKQAS UPDATE CPT MODIFIERS] - This option has had it's DIC(w) value 
 updated to call ACKQUTL8.
  
 ACKQUTL5 - This routine has been enhanced to call ACKQUTL8.
 ACKQUTL4 - This routine has been enhanced to call ACKQUTL8.
  
 8) Display the correct version of the CPT code description
 =================================================
  
 Enter Cost Information for Procedures [ACKQAS COST ENTRY]
  
 The CPT code text displayed within the Enter Cost Information for 
 Procedures [ACKQAS COST ENTRY] option will be valid for the operation
 based on the current date.
  
 ACKQAS3 - This routine has been enhanced to call ACKQUTL8.

 This Patch was created with Kernel V8 KIDS.  Follow these instructions:
  
 1) Use the 'INSTALL/CHECK MESSAGE' option on the PackMan menu to load the
 KIDS patch onto your system.
  
 2) If any of the routines listed in the ROUTINE SUMMARY section are mapped
 at your site, remember to disable mapping before installing the patch and
 to re-enable mapping when you are finished.
  
 3) This patch should be installed during off peak hours when the
 Audiology and Speech Pathology users are off the system.  The estimated
 installation time is two (2) minutes.
  
 4) Using the Kernel Installation and Distribution System menu (KIDS), you
 may choose the following options prior to installing this patch:
  
   a)  Backup a Transport Global - this option will create a backup message
       of any routines exported with the patch.  It will NOT backup any
       other changes such as data dictionaries or templates.
  
   b)  Compare Transport Global to Current System - this option will allow
       you to view all changes that will be made when the patch is
       installed.  It compares all components of the patch (routines, DDs,
       templates, etc.)
  
   c)  Verify Checksums in Transport Global - this option will allow you to
       ensure the integrity of the routines that are in the transport
       global.
  
   d)  Print Transport Global - this option will allow you to view the
       components of the KIDS build.
  
 5) Use the 'Install Package(s)' option under the 'Installation' menu.
  
 6) When prompted 'Want KIDS to INHIBIT LOGONs during the install? YES//
 respond "NO".

 Protocols ?' respond "NO".

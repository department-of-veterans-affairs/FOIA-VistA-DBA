Description: 
============

 This patch fixes miscellaneous Health Summary errors:
 
     FIX                                                    REFERENCE
 ------------------------------------------------------------------------------
 1.  Health Summary Protocols no longer re-prompt for       NOIS CHS-0698-42562
     patient selection if a patient was previously          NOIS NAS-1198-30117
     selected.  Additionally, the entry action to the
     corresponding Health Summary Options have been 
     changed to force patient selection.
                                                        
       Routines:  GMTS, GMTSADHC, and GMTSDVR
 
       Options:   GMTS HS ADHOC
                    Ad Hoc Health Summary
 
                  GMTS HS BY PATIENT
                    Patient Health Summary
 
                  GMTS HS BY PATIENT & DATE RANG
                    Range of Dates Patient Health Summary
 
                  GMTS HS BY PATIENT & VISIT
                    Visit Patient Health Summary
    
 2.  The default occurence limit for Compensation/Pension   NOIS MIW-1097-40242
     has been set to 99999 to allow printing of C&P data
     even when the user deletes the occurence limit.
 
        Routines:  GMTSAMIE
 
 3.  ADR Adverse Reactions/Allergies component has been     NOIS FAV-1098-70005
     fixed to print "No Known Allergies" (NKA) when         NOIS HIN-0998-41543
     applicable.                                            NOIS ISL-1198-52148
                                                            NOIS WLA-0898-60440
        Routines:  GMTSALG and GMTSALGB                     NOIS WNY-1098-10905
                                                            NOIS WPB-0998-30097
 
 4.  DC'd Lab orders are now being reported with a status   NOIS IOW-1197-40984
     of "Canceled" instead of "Ordered"
 
        Routines:  GMTSLROE
 
 5.  Variables from a previous lookup have been cleaned     NOIS BUT-0598-20476
     up allowing jumping from one component to another      NOIS REN-0698-61055
     component on the first screen display.                 NOIS SYR-0398-10648
                                                            NOIS SYR-1098-11448
        Routines:  GMTSPNB and GMTSUP
              
 6.  Variables from a previous lookup have been cleaned     NOIS CMO-0598-41722
     up fixing the Undefined Variable ^GMT(142,-1,99) 
     error at EN+2^XQORM
 
        Routines:  GMTSPNB
 
 7.  Variables from a previous lookup have been cleaned     NOIS CAH-0298-30921
     up fixing the Undefined Variable DFN at 
     RJUMP+6^GMTSUP1
 
        Routines:  GMTSUP1
 
 8.  Blank lines on the second setion of the Vital Sign     NOIS WRJ-0698-11966
     are suppressed.  If no data is found for the second
     section, a single line stating "No Data" will be
     displayed.
 
        Routines:  GMTSVS and GMTSVSS
 
 9.  All dates resolve to a 4 character year                Y2K
 
 10. Direct references to the ^ICPT global have been        Delete DBIA 1274
     replaced with supported APIs to the CPT package.
 
        Routines:  GMTSRO and GMTSRON
 
 11. The following routines are released to include         NDBI Team
     changes to support NDBI modifications required at
     integrated sites to access legacy data collected
     via the NDBI Health Summary Utility.  The routines
     will check to see if the site is integrated and if
     yes, the collected legacy data willbe merged with
     the current data in Health Summary displays.  This
     patch makes it NO LONGER NECESSARY for integrated
     sites to track these routines and have the NDBI 
     team send the amended NDBI changes every time a
     Health Summary patch is released.
 
        Routines:  GMTSDA, GMTSDGA, GMTSDGH, GMTSDGP, 
                   GMTSLRA, GMTSLRC, GMTSLRCP, GMTSLRM,
                   GMTSLRMB, GMTSLRS, GMTSLRS7, GMTSLRSC,
                   GMTSLRSE, GMTSLRT, GMTSPSO, GMTSSOWK
                   and GMTSU
  
  INFORMATIONAL:
  =============
   
 Documentation Changes
 
 1.  Historical Encounters       STL-0397-40498  Install Guide
 
       Several sites have reported that they are unable to view historical
       encounters which occurred before the PCE Parameter for HEALTH SUMMARY
       START DATE.  To view earlier encounters, you will need to use Fileman
       to edit the HEALTH SUMMARY START DATE field #.03 in the PCE PARAMETER
       file #815, and change the date to a date which would include the
       historical encounters.  This change should not affect the displaying
       of data by PCE.  A note to this effect should be placed in the
       Patient Care Encounter (PCE) and Visit Tracking Installation Guide,
       in the section entitled "Post-Installation -- PCE and Visit Tracking
       Set-Up"
 
 2.  Principal Anesthesiologist  WPB-1198-30668  GMTS*2.7*11 Patch Description
  
       In the description of patch GMTS*2.7*11, it states that the "SURGERY
       REPORTS component to include the PRINCIPAL ANESTHESIOLOGIST in the
       report.  The reference to "PRINCIPAL ANESTHESIOLOGIST" should be 
       changed to read "PRINCIPAL ANESTHETIST"

  INSTALLATION:
  =============
  
    This patch can be installed with users on the system, however it is
    recommended that installation occur durring low-peak use.  
    Installation will take less than five minutes.  A majority of this
    time will be spent rebuilding the Menu Trees.
    
    1.  Use the INSTALL/CHECK MESSAGE option on the PackMan menu, to
        load the KIDS Transport Distribution global.
    
    2.  Review your mapped set.  If any of the routines listed in
        the ROUTINE SUMMARY section are mapped, they should be
        removed from the mapped set at this time.
    
    3.  From the Kernel Installation and Distribution System (KIDS) Menu,
        select the Installation menu.
      
    4.  From the Install menu, you may elect to use the following
        options (when prompted for INSTALL NAME, enter GMTS*2.7*28):
     
        a.  Verify Checksums in Transport Global
        b.  Print Transport Global
        c.  Compare Transport Global to Current System
        d.  Backup a Transport Global
     
    5.  Use the Install Package(s) option and select the package
        GMTS*2.7*28.
  
    6.  When prompted "Want KIDS to Rebuild Menu Trees Upon Completion of
        Install? YES//", respond "YES"
 
    7.  When prompted "Want to DISABLE Scheduled Options, Menu Options, 
        and Protocols? YES//", respond "YES."  
 
        Disable the following options:
 
        GMTS HS ADHOC                  Ad Hoc Health Summary
        GMTS HS BY PATIENT             Patient Health Summary
        GMTS HS BY PATIENT & DATE RANG Range of Dates Patient Health Summary
        GMTS HS BY PATIENT & VISIT     Visit Patient Health Summary
    
        Disable the following protocols:
 
        GMTS HS BY PATIENT & DATE      Range of Dates Patient Health Summary
        GMTS HS BY PATIENT & VISIT     Visit Patient Health Summary
 
 
    8.  If routines were unmapped as part of step 2, they 
        should be returned to the mapped set once the installation
        has run to completion.

Subject: OFFICER NAME ON VA POLICE DAILY OPERATIONS JOURNAL

Category: 
  - Routine
  - Print Template

Description:
============

 Modify the PDJ Print Daily Journal option which prints the VA POLICE 
 DAILY OPERATIONS JOURNAL, to display the officer's names as 
 lastname,finit (finit=first name initial) to distinguish officers with
 the same last name.
   
 ASSOCIATED NOIS:
 ================

                 OPERATIONS JOURNAL

 NOIS OVERVIEW
 =============
   
 TOM-0504-41551  Modify software to include 1st name of officer on DAILY 
                 OPERATIONS JOURNAL
   
 1.  Problem
      -----------
    
     When printing the DAILY OPERATIONS JOURNAL the officer's names
     appear as the first 9 characters of the last name only. The site
     wants to see the first 5 characters of last name and the first
     character of first name and middle initial:  aaaaa,b c
    
          
     Resolution
     ---------------
          
     A modification was made to the routine ESPJOU to print officer's names
     as first 5 characters of last name and first character of first name and
     middle name:   BECKW,R G  
     
 2.  Problem
      -----------
     
     When printing the OFFENSE MATCH REPORT the officer's names
     appear as the first 9 characters of the last name only. The site
     wants to see the first 5 characters of last name and the first
     character of first name and middle initial:  aaaaa,b c
     
        
     Resolution
     ---------------
       
     A modification was made to the Print Template [ESP OFFENSE MATCH 
     REPORT  FILE #912] to print officer's names as first 5 characters
     of last name and first character of first name and middle name:
      BECKW,R G 
     
 3.  Problem
      -----------
      
     When printing the CASE ASSIGNMENT REGISTERS the officer's names
     appear as the first 9 characters of the last name only. The site
     wants to see the first 5 characters of last name and the first
     character of first name and middle initial:  aaaaa,b c
      
        
     Resolution
     ---------------
       
     A modification was made to the Print Template [ESP CASE ASSIGNMENT 
     REGISTER  FILE #912] to print officer's name as first 5 characters
     of last name and first character of first name and middle name:
      BECKW,R G 
      
 4.  Problem
      -----------
      
     The following options print a bogus page feed on the home device prior
     to the report printing:  FOLLOW-UP ENTRY ADD/EDIT, PRINT ANY
     FOLLOW-UP NOTE, PRINT ANY OFFENSE REPORT and REVIEW
     COMPLETED OFFENSE REPORT.
      
          
     Resolution
     ---------------
      
     A modification was made to the routines ESPORP and ESPOFP to 
     skip the initial page feed for these reports when directing to the home
     device.

 INSTALLATION INSTRUCTIONS
 =========================
   
   This patch can be installed at any time.  Users do not need to be off
   the system.  Installation will take less than 2 minutes.
   
  1.  Choose the PackMan message containing this patch and invoke the 
      INSTALL/CHECK MESSAGE PackMan option.
   
  2.  (DSM for Open VMS sites only) Disable routine mapping on all systems
      for the routines listed in the Routine Summary.
    
  3.  Start up the Kernel Installation and Distribution System Menu 
      [XPD MAIN]: 
     
           Edits and Distribution ...  
           Utilities ...  
           Installation ...  
     
      Select Kernel Installation & Distribution System Option: INStallation
                                                                ---
                                                                 
           Load a Distribution 
           Print Transport Global 
           Compare Transport Global to Current System 
           Verify Checksums in Transport Global 
           Install Package(s) 
           Restart Install of Package(s) 
           Unload a Distribution 
           Backup a Transport Global  
   
  4.  From this menu, you may elect to use the following options
      (when prompted for the INSTALL NAME, enter ES*1.0*42):
   
      a.  Backup a Transport Global - This option will create a backup
          message of any routines exported with this patch. It will not
          backup any other changes such as DD's or templates.
      b.  Compare Transport Global to Current System - This option will
          allow you to view all changes that will be made when this patch
          is installed.  It compares all components of this patch
          (routines, DD's, templates, etc.).
      c.  Verify Checksums in Transport Global - This option will allow
          you to ensure the integrity of the routines that are in the
          transport global.
   
  5.  Use the Install Package(s) option and select the package ES*1.0*42.
   
      a.  When prompted 'Want KIDS to INHIBIT LOGONs during the install?
          YES//', answer NO (unless otherwise indicated).
      b.  When prompted 'Want to DISABLE Scheduled Options, Menu Options,
          and Protocols? YES//', answer YES
           1.  ESP OFFENSE MATCH REPORT
           2.  ESP CASE ASSIGNMENT REGISTER (ALL)
           3.  DAILY OPERATIONS JOURNAL
           4.  PRINT ANY FOLLOW-UP NOTE
           5.  PRINT ANY OFFENSE REPORT
           6.  REVIEW COMPLETED OFFENSE REPORT
           7.  ESP FOLLOW-UP ENTRY ADD/EDIT
    
      c.  When prompted 'Enter protocols you wish to mark as 'Out Of 
          Order':', press <return>.
   
  6.  (DSM for Open VMS sites only) Optional - Include the routines
      distributed with this patch in the mapped routine set.
      
      NOTE: This step is only necessary if you performed step 2 or if
      you wish to include the routines in your mapped set.

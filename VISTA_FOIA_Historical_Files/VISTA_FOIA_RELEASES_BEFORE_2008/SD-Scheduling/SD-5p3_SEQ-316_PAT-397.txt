Subject: UP ARROW AT INSTITUTION PROMPT DELETES SERV/SPEC ENTRY

Category: 
  - Routine

Description:
============

 This patch will correct the '^' deletion problem when entering
 Wait List Parameters for Service Specialty.  
  
 Also, the patch will correct the Service/Specialty pointer
 problem found when entering Service/Specialty in Wait
 List Enter/Edit and seeing a different Service/Specialty
 appear in inquiry, reporting or editing a previous entry.

 NOIS OVERVIEW:
 ==============
  
 Problem:  UNY-0903-10519  UPARROW AT INSTITUTION DELETES SERV/SPEC ENTRY
 ------------
   
 When editing a Service/Specialty in the Wait List Edit Parameter 
 option, entering an '^' at the 'INSTITUTION' prompt deletes the
 entire Service/Specialty entry.
   
 Resolution:
 ---------------
  
 Modify routine SDWLPE to handle the '^' escape and only delete the entry 
 if no previous institutions entered.   
   
  
 Problem:  FGH-0703-30922   WAIT LIST ASSIGNMENT PROBLEMS
 ------------
   
 When entering a Service/Specialty in the Wait List Enter/Edit 
 option, upon certain conditions the entry will point to a Service/
 Specialty different from the original entry.
    
 Resolution:
 ---------------
   
 Modify routine SDWLE2 to save the correct pointer through Fileman
 found in the Service Specialty File #409.31.

   the system.  Installation will take less than 2 minutes.
   
  1.  Choose the PackMan message containing this patch and invoke the 
      INSTALL/CHECK MESSAGE PackMan option.
   
  2.  (DSM for Open VMS sites only) Disable routine mapping on all systems
      for the routines listed in the Routine Summary.
    
  3.  Start up the Kernel Installation and Distribution System Menu 
      [XPD MAIN]: 
     
           Edits and Distribution ...  
           Utilities ...  
           Installation ...  
     
      Select Kernel Installation & Distribution System Option: INStallation
                                                                ---
                                                                 
           Load a Distribution 
           Print Transport Global 
           Compare Transport Global to Current System 
           Verify Checksums in Transport Global 
           Install Package(s) 
           Restart Install of Package(s) 
           Unload a Distribution 
           Backup a Transport Global  
   
  4.  From this menu, you may elect to use the following options
      (when prompted for the INSTALL NAME, enter SD*5.3*397):
   
      a.  Backup a Transport Global - This option will create a backup
          message of any routines exported with this patch. It will not
          backup any other changes such as DD's or templates.
      b.  Compare Transport Global to Current System - This option will
          allow you to view all changes that will be made when this patch
          is installed.  It compares all components of this patch
          (routines, DD's, templates, etc.).
      c.  Verify Checksums in Transport Global - This option will allow
          you to ensure the integrity of the routines that are in the
          transport global.

          ENTER/EDIT WAIT LIST (SCH/PCMM     SD WAIT LIST ENTER/EDIT
  
      c.  When prompted 'Enter protocols you wish to mark as 'Out Of 
          Order':', press <return>.
   
  6.  (DSM for Open VMS sites only) Optional - Include the routines
      distributed with this patch in the mapped routine set.
      
      NOTE: This step is only necessary if you performed step 2 or if
      you wish to include the routines in your mapped set.

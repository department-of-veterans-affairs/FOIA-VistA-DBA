Subject: GUI RELATED APIS FOR ALERTS

Category: 
  - Routine
  - Data Dictionary
  - Enhancement (Mandatory)
  - Other

Description: 
============

 Currently, alert processing in applications such as CPRS is handled by the
 applications which generate the alerts.  This results in some alerts which
 may be able to be handled within one available client application, not
 being available in other applications which the user may be using.  In
 addition, some alerts have processing which is only available on the
 server in roll and scroll mode.
  
 This patch adds support for inter-application alert processing in client
 applications by providing application programming interfaces (API) related
 to graphical user interface (GUI) programming and support for Kernel
 components including those related to alert processing.
  
 In addition, it adds a LONG INFO TEXT field and a GUID FOR GUI fields to
 the Alert file.  The LONG INFO TEXT field will permit informational text
 of any length to be passed with an alert.  When the alert is selected, the
 contents of the LONG INFO TEXT field will be displayed, in a Screenman
 form within the roll & scroll environment, and within a text display box
 within the GUI environment.
  
 The GUID FOR GUI adds an interface GUID (a 32 character string containing
 hexadecimal digits in a specific format within curly braces) to permit a
 program on the client to process the alert.  The presence of a GUID
 indicates that the alert can be processed within a GUI environment, and is
 used to open the correct application to process the alert within the GUI
 environment.
  
 The patch contains two major parts.  The server installation build is
 included in the patch module.  The client-related tools for development of
 applications based on the Kernel Delphi Components are packed as an
 executable installation.  This module, KDC1_0.exe can be obtained from
 the NVS anonymous directories.
  
 The fields GUID FOR GUI (#3.01) and LONG INFO TEXT (#4) are added to the
 ALERT (#8992) and ALERT TRACKING (#8992.1) files.
  
 The build includes one Broker (Client/Server) option [XQAL GUI ALERTS].
  
 The build includes one Remote Procedure file entry [XQAL GUI ALERTS].  The
 XQAL GUI ALERTS Remote procedure is necessary for processing of alerts
 with the Kernel Delphi Components.
  
 The build includes four routines.  XQALDATA, XQALSET and XQALDOIT are
 updated for the XQAGUID and XQATEXT (Long Text) variables.  XQALGUI is a
 new routine which does the processing for the XQAL GUI ALERTS remote
 procedure.

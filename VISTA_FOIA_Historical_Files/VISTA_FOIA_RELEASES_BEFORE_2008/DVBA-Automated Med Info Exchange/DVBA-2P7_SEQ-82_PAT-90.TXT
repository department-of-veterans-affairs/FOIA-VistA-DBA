Description:
============

  This patch provides VistA updates necessary to support the changes
  made to the Compensation and Pension Records Interchange (CAPRI)
  Graphical User Interface (GUI)enhancements carried in upcoming
  patch DVBA*2.7*91.
  
  
  Compliance Date
  ---------------
   21 Days - To facilitate release of CAPRI GUI DVBA*2.7*91
   Users need DVBA*2.7*90 in place before
   New GUI DVBA*2.7*91 is placed on desktops
   which is schedule for release in less than 30 days.
  
  
  E3R(s) addressed by this patch
  ------------------------------
  There are no E3Rs associated with this patch.
  
  
  NOIS call(s) addressed by this patch
  ------------------------------------
  There are no NOIS calls associated with this patch.
  
  
  Changes to CAPRI TEMPLATES FILE (#396.17)
  =========================================
   * The IN USE field (#17) is added to store the
     name OF THE user who is editing the template.
   * The FORM SAVE BUFFER multiple (#8.5) is added to store 
     various saVe points from which changes may be restored.
  
  
  OPTION modified by this patch
  =============================
   * DVBA CAPRI GUI - RPCs listed below are "registered", due
                      to there inclusion on the RPC multiple.
  
 
  REMOTE PROCEDUREs added by this patch
  =====================================
   * DVBAB FETCH 1U4N - enables the CAPRI GUI to get Social Security
                        Numbers in U1N4 format.
   * DVBAB FIND DUPS - enables the CAPRI GUI to find potential
                       patient matches prior to adding a new patient.
   * DVBAB FORM COPY - enables the CAPRI GUI to copy one
                       template's data into another template.
   * DVBAB FORM DATA BACKUP - enables the CAPRI GUI to save redundant
                              copies of a CAPRI session date in an
                              effort to alleviate potential data loss.
   * DVBAB FORM DATA BACKUP DELETE - enables the CAPRI GUI to delete
                                     redundant copies after "co-signing"
   * DVBAB FORM DATA BACKUP RESTORE - enables the CAPRI GUI to restore
                                      data from one of the redundant saves.
   * DVBAB NEW PERSON FILE - enables the CAPRI GUI to get data from file 
 #200.
   * DVBAB SET DIVISION - enables the CAPRI GUI to set the division
                          as a work-around with a known issue in the broker.
   * DVBAB TEMPLATE REPORT FULL - enables the CAPRI GUI to see the status
                                  of templates over some time frame.

  INSTALLATION INSTRUCTIONS
  =========================
  This patch can be installed with users on the system, however it should
  be installed during off hours to minimize disruption to users.
  
  Installation should take less than 5 minutes.
  
  NOTE: The following is a list of REQUIRED builds for this KIDS 
  distribution. KIDS will not allow the installation of this patch
  without the prior installation of:
   
            DVBA*2.7*35    DVBA*2.7*42    DVBA*2.7*53    DVBA*2.7*57
  
  
  1.  Use the LOAD A DISTRIBUTION option on the KIDS INSTALLATION Menu,
      and enter DVBA*2.7*90.KID.
  
  2.  From the Kernel Installation and Distribution System Menu,
      select the Installation menu.
  
  3.  Review your mapped set.  If any of the routines listed in the
      ROUTINE SUMMARY section are mapped, they should be removed
      from the mapped set at this time.
  
  4.  From the Kernel Installation and Distribution System Menu,
      select the Installation menu.
  
  5.  From this menu, you may elect to use the following options
      (when prompted for INSTALL NAME, enter DVBA*2.7*90):
  
        a.  Backup a Transport Global - this option will create a backup
            message of any routines exported with the patch.  It will
            NOT backup any other changes such as DDs or templates.
        b.  Compare Transport Global to Current System  - this option
            will allow you to view all changes that will be made when
            the patch is installed.  It compares all components of the
            patch (routines, DDs, templates, etc.).
        c.  Verify Checksums in Transport Global - this option will
            allow you to ensure the integrity of the routines that
            are in the transport global.
        d.  Print Transport Global - this option will allow you to
            view the components of the KIDS build.
   
  6.  Use the Install Package(s) option and select the package DVBA*2.7*90.
  
  7.  When prompted 'Want KIDS to Rebuild Menu Trees Upon Completion of
      Install? YES//', respond NO.
  
  8.  When prompted 'Want KIDS to INHIBIT LOGONs during the install?
      YES//', respond NO.
  
  9.  When prompted 'Want to DISABLE Scheduled Options, Menu Options, and
      Protocols? YES//', respond NO.
  
  10. If routines were unmapped as part of step 3, they should be returned
      to the mapped set once the installation has run to completion.

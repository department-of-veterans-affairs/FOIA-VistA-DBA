Subject: PROTOCOL CHANGES

Category: 
  - Other

Description: 
============

  Required Builds:  Master Patient Index Vista 1.0 - which is a controlled
  release.  If you do not have Master Patient Index Vista, contact your 
  CIRN Implementation Manager about getting on the installation schedule.
   
  
  Patch HL*1.6*57 added a Subscriber multiple field to the Protcol (#101)
  file.  MPIF protocols were exported prior to HL*1.6*57 and did not have
  the Subscriber field.  
  
  The following protocols are being sent:
  
         Protocol                        Subscriber
         --------                        ----------
   MPIF A28 REQUEST                  MPIF A28 RESPONSE
   MPIF A29 SERVER                   MPIF A29
   MPIF A30 SERVER                   MPIF A30
   MPIF CMOR COMPARISON SERVER       MPIF CMOR COMPARISON CLIENT
   MPIF CMOR REQUEST                 MPIF CMOR RESPONSE
   MPIF CMOR RESULT SERVER           MPIF CMOR RESULT CLIENT
   MPIF ICN-Q02 SERVER               MPIF TEST
  
  
  
  **Note:  There are no routines associated with this patch,
           just Protocols (#101).
  
  TEST SITE:  Bay Pines VAMC

  INSTALLATION INSTRUCTIONS:
  ==========================
  
   This patch should be loaded during non-peak hours to minimize disruption
   to the users and the processing of HL7 messages.
  
   This patch will take less than 5 minutes to install.
  
   1.  Users are allowed on the system during patch installation.
   
   2.  Use the INSTALL/CHECK MESSAGE option on the PackMan menu. [Note:
       TEXT PRINT/DISPLAY option in the PackMan menu will display the patch
       text only.
  
   3.  TaskMan does not need to be stopped or placed in a wait state. 
  
   4.  From the Kernel Installation and Distribution System Menu, select
       the Installation menu.  
  
   5.  From this menu, you may elect to use the following options:
       a.  Backup a Transport Global - this option will create a backup
           message of any routines exported with the patch.  It will NOT
           backup any other changes such as DDs or templates.
       b.  Compare Transport Global to Current System  - this option will
           allow you to view all changes that will be made when the patch
           is installed.  It compares all components of the patch
           (routines, DDs, templates, etc.).
       c.  Verify Checksums in Transport Global - this option will allow
           you to ensure the integrity of the routines that are in the
           transport global.
       d.  Print Transport Global - this option will allow you to view the
           components of the KIDS build.
  
   6.  Use the Install Package(s) option and select the package MPIF*1*4.
  
   7.  When prompted 'Want to DISABLE Scheduled Options, Menu Options, and
       Protocols? YES//', respond Yes. The following Protocols should be
       placed out of Order:
         MPIF A28 RESPONSE
         MPIF A29
         MPIF A30
         MPIF CMOR COMPARISON CLIENT
         MPIF CMOR RESPONSE
         MPIF CMOR RESULT CLIENT
         MPIF TEST
         MPIF ICN-Q02 SERVER

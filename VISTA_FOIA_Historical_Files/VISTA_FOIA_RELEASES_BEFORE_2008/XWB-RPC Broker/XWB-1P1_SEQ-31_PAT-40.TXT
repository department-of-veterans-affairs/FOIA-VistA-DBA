Description:
============

 This is a client-side only patch for the Broker Development Kit. 
 The installation of this patch is only necessary for sites doing Delphi 
 development of RPC Broker applications and then only on the development 
 workstations.
  
 It provides for (in conjunction with server-side patch XWB*1.1*35) a non-
 callback or UCX connection which permits connection through firewalls.  It
 also provides support for a single sign-on based on CCOW (Clinical 
 Context Object Workgroup) User Context with the TCCOWRPCBroker component, 
 which has all of the functionality of the TRPCBroker component and adds 
 CCOW User Context.  The patch provides support for Delphi 5 through Delphi 7.
  
 Prior to installation on the developer's workstation, it is recommended
 that you rename the BDK32 directory to something such as BDK32_OLD or
 BDK32_050114 to keep the prior version intact.
  
 To install run the installation program XWB1_1P40PG.exe.  It is recommended 
 that you accept the default directory (C:\Program Files\Vista\BDK32) for
 installation.  Check the versions of Delphi that you want installed (this 
 requires that the versions be present on the system).  Installation will
 then continue until finished (about one minute).
  
 After installation, to correctly activate the help file, start a version
 of Delphi and drop an instance of the TRPCBroker from the Kernel tab onto
 a form.  Select the TRPCBroker instance and click on a property such as
 'Server' in the Object Inspector.  At that point press the F1 key to
 activate context sensitive online help (If you do this first on the form,
 it will not activate correctly in Delphi 6 or Delphi 7).
  
  
 The installation requires less than 20 Megabytes of disk space on a 
 workstation.
  
 The patch priority is Mandatory (since there is no choice for optional)
  
 The patch type is: Other and Enhancement

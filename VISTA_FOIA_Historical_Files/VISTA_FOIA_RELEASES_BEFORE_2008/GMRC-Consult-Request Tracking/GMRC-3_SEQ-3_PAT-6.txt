Subject: FIX TO ALLOW SERVICE NAMES TO BE CHANGED

Category: 
  - Routine

Description: 
============

 This patch fixes an error that occurs when the name of an entry in the
 REQUEST SERVICES (#123.5) file is changed. 
   
 Changes were made to the kill and set logic called by the AC1
 cross-reference for the SERVICE NAME (.01) field of the REQUEST SERVICES
 file (123.5). The logic is in the routine GMRCDDX.
  
  NOIS CALLS ADDRESSED:
  ====================
  PUG-0598-50660    M error after name change of cons service
  AMA-0598-71719    <UNDEFINED>EN+10^GMRCSRVS
  CHA-0998-31113    cross-references not deleted 
  CHY-1198-52019    <UNDEFINED>PROC+5~GMRCASV
  SPO-1298-51162    <UNDEFINED>PROC+5~GMRCASV AFTER CONSULT D
  LAS-0199-61755    Undefined error when file 123.5 subfield
  SYR-0299-10933    CANNOT ACCESS CONSULT PKG
  SLC-0299-50982    double entry of services

Subject: CCPC PATIENT STATEMENT ENHANCEMENTS

Category: 
  - Routine
  - Input Template
  - Enhancement (Mandatory)
  - Data Dictionary

Description:
============

 This patch contains two enhancements and resolves one Remedy ticket.
  
 The Chief Business Office has requested additional enhancements to the
 software that transmits the Patient Statements from the VAMC to the AAC.
  
  
 Description of enhancements:
 ----------------------------
 1. Remove the "In Part" and "In Full" wording from the patient statements.
  
 2. Identify patients to the AAC that require larger print (font) on their
    billing statement.  A new prompt has been added to option ADDRESS DISPLAY/
    EDIT [PRCA VEN BIL].  
    
    NEW PROMPT => LARGE PRINT NEEDED ON STMT: NO//
  
  
 This patch addresses the following New Service Request (NSR):
 -------------------------------------------------------------
 New Service Request (NSR) 2030604
  
  
 This patch addresses the following Remedy Ticket(s):
 ----------------------------------------------------
 1. HD0000000063287 CCPC Failed Trans. Sta. #460
  
  
 Overview of Remedy Ticket(s):
 -----------------------------
 1. HD0000000063287 CCPC Failed Trans. Sta. #460
  
  
 Problem:
 --------   
 Special characters in the patients address caused a transmission
 failure of the billing statements to the AAC (Austin Automation Center).
  
 The following special characters are the cause of the problem:
 1. Tilde ( ~ )
 2. Pipe  ( | )
  
  
 Resolution:
 -----------
 If the patients address contains either a tilde or pipe symbol, it is
 removed from the address for transmission to the AAC.

   ROUTINE: PRCA223
   
   DESCRIPTION
   -----------
   This routine updates the entries in the AR DEBTOR file (#340) with
   a value of zero for the LARGE PRINT NEEDED ON STMT field (#111).
   Once this process is completed, all 1st party debtors will have a
   default value of NO for the prompt LARGE PRINT NEEDED ON STMT.

Subject: CMOR SCORE CALCULATION STATUS OPTION

Category: 
  - Routine
  - Enhancement (Mandatory)

Description: 
============

  CMOR Score Calculation Status option [RGCIRN CMOR STATUS]
  A change was made in routine RGVCCMR1 to provide an accurate count
  of the number of patients with CMOR (CIRN Master of Record) scores.
  Also, as per NOIS BIL-0599-30258, the wording of the display has
  been refined.  When the status is "SUCCESSFULLY COMPLETED", the
  verbiage regarding the number of processed records has been removed.
  When this option is utilized, it now shows the following dialogue.
   
    The CIRN CMOR Activity Score Generator
  
    ..one moment please...
  
    There are 324795 records in your PATIENT file.
    The last record number is 398257.
    Last Patient Processed: MOUSE,MINNIE  SSN: 123123123  [RECORD# 393043]
   
    The CMOR score initialization last started on Apr 05, 1999 3:30:04 pm.
    Status: SUCCESSFULLY COMPLETED on Apr 08, 1999 6:15:19 am.
    CMOR Score Count: 67107

  Installation Instructions:
  ==========================
  This patch may be loaded with users on the system.  You may wish
  to install it during non-peak hours.  Installation will take less
  than 1 minute.
   
  1.  Use the INSTALL/CHECK MESSAGE option on the Packman menu.
      [Note: TEXT PRINT/DISPLAY option in the PackMan menu will display
      the patch text only.]
   
  2.  None of the routines in the CLINICAL INFO RESOURCE NETWORK package
      have been recommended for routine mapping.
   
  3.  From the Kernel Installation and Distribution System Menu, select
      the Installation menu.
   
  4.  From this menu, you may elect to use the following options (when
      prompted for the INSTALL NAME, enter RG*1.0*2).
   
      (a)  Backup a Transport Global - this option will create a backup
           message of any routines exported with the patch.  It will NOT
           backup any other changes such as DDs or templates.
   
      (b)  Compare Transport Global to Current System - this option will
           allow you to view all changes that will be made when the
           patch is installed.  It compares all components of the patch
           (routines, DDs, templates, etc.).
   
      (c)  Verify Checksums in Transport Global - this option will allow
           you to ensure the integrity of the routines that are in the
           transport global.
   
      (d)  Print Transport Global - this option will allow you to view the
           components of the KIDS build.
   
  5.  Use the Install Package(s) option and select the package RG*1.0*2.
   
  6.  When prompted 'Want to DISABLE Scheduled Options, Menu Options, and
      Protocols? YES//', respond NO.
   
  7.  Routine RGP2ENV can be deleted from your system after successful
      installation.

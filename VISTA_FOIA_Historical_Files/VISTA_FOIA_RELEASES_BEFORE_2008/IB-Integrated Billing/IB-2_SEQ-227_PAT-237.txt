Description:

 ERROR message.

 NOIS Overview:
 -------------
 1. ASH-0203-31514     IB Bulletin Lacks Patient Info To Act On
  
    Problem:
    -------
    An INTEGRATED BILLING BACKGROUND ERROR bulletin is generated
    when the IB Back Billing job fails to lock the IB PATIENT COPAY
    ACCOUNT (#354.7) file.  This bulletin serves no purpose at the
    moment because it lacks the patient information needed to follow
    up on the error.  The current bulletin looks like:
  
 Subj: INTEGRATED BILLING BACKGROUND ERROR  [#6200] 07/30/03@12:44
 10 lines
 From: INTEGRATED BILLING PACKAGE  In 'IN' basket.   Page 1  *New*
 ---------------------------------------------------------------------------
 Processing of entries in Integrated Billing has
 been suspended while canceling:
  
  
 You should determine if these co-payments have been passed to
 Accounts Receivable.
 The following error(s) was encountered:
  
     Unable to lock for back billing job.
  
  
 Enter message action (in IN basket): Ignore//
  
    Resolution:
    ----------
    The INTEGRATED BILLING BACKGROUND ERROR bulletin generated when
    the IB Back Billing job fails to lock the IB PATIENT COPAY
    ACCOUNT (#354.7) file has been modified to include the patient
    name, SSN, and the IB reference number(s) involved.  The report
    will now look like:
  
 Subj: INTEGRATED BILLING BACKGROUND ERROR  [#767741] 07/30/03@00:47
 10 lines
 From: INTEGRATED BILLING PACKAGE  In 'IN' basket.   Page 1
 ----------------------------------------------------------------------------
 Processing of entries in Integrated Billing has
 been suspended while canceling:
  
  
 You should determine if these co-payments have been passed to
 Accounts Receivable.
 The following error(s) was encountered:
  
     Unable to lock the IB PATIENT COPAY ACCOUNT (#354.7) file for back 
 billing job related to INGRAHAM,WAYNE (002021948) and IB reference number(s)
 : 4234, 338.
  
  
 Enter message action (in IN basket): Ignore//

 Installation Instructions
 =========================
 Routine IBARX is called by several Outpatient Pharmacy routines.
 Before installing this patch, use List Tasks [XUTM INQ] option on the
 Taskman Management [XUTM MGR] menu to list currently running tasks.
 Do not install this patch while Outpatient Pharmacy users are using
 the software.  Installation will take less than 2 minutes.
   
  1.  Choose the PackMan message containing this patch and invoke the 
      INSTALL/CHECK MESSAGE PackMan option.
  
  2.  (DSM for Open VMS sites only) Disable routine mapping on all systems
      for the routines listed in the Routine Summary.
   
      NOTE:  If the routines included in this patch are not currently
      in your mapped routine set, please skip this step.
   
  3.  Start up the Kernel Installation and Distribution System Menu 
      [XPD MAIN]: 
    
           Edits and Distribution ...  
           Utilities ...  
           Installation ...  
    
      Select Kernel Installation & Distribution System Option: INStallation
                                                               ---
                                                                
           Load a Distribution 
           Print Transport Global 
           Compare Transport Global to Current System 
           Verify Checksums in Transport Global 
           Install Package(s) 
           Restart Install of Package(s) 
           Unload a Distribution 
           Backup a Transport Global  
  
  4.  From this menu, you may elect to use the following options
      (when prompted for the INSTALL NAME, enter IB*2.0*237):
  
      a.  Backup a Transport Global - This option will create a backup
          message of any routines exported with this patch. It will not
          backup any other changes such as DD's or templates.
      b.  Compare Transport Global to Current System - This option will
          allow you to view all changes that will be made when this patch
          is installed.  It compares all components of this patch
          (routines, DD's, templates, etc.).
      c.  Verify Checksums in Transport Global - This option will allow
          you to ensure the integrity of the routines that are in the
          transport global.
  
  5.  Use the Install Package(s) option and select the package IB*2.0*237.
  
      a.  When prompted 'Want KIDS to INHIBIT LOGONs during the install?
          YES//', answer NO (unless otherwise indicated).
      b.  When prompted 'Want to DISABLE Scheduled Options, Menu Options,
          and Protocols? YES//', answer NO.
  
  6.  (DSM for Open VMS sites only) Optional - Include the routines
      distributed with this patch in the mapped routine set.
     
      NOTE: This step is only necessary if you performed step 2 or if
      you wish to include the routines in your mapped set.

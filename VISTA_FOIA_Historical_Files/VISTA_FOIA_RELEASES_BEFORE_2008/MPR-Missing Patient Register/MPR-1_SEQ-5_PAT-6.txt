Subject: REMOVE MISSING PATIENT REGISTER FROM VISTA

Category: 
  - Other

Description:
============

 *********************** NOTE********************************************
 Before installing patch, check the following:
  
 In DSM, make sure Access Privileges for the global ^MPR are as follows:
 System <RWP>      World <RWP>       Group <RWP>       UCI <RWP> 
   
 In Cache, make sure global properties for ^MPR are as follows: 
 Owner=RWD         Group=RWD         World=RWD         Network=RWD
 *************************************************************************
 This patch will remove the Missing Patient Register (MPR) from the 
 Vista system.  This includes Data files, dictionaries, routines,
 templates, mailgroups, options and bulletins.

 NOIS OVERVIEW:
 ==============
  
 Problem:
 --------
  
 This package is obsolete and has been replaced by a Web based product
 and NOIS ISD-1102-71711 is the request for removal of the old system.
  
 Resolution:
 ===========
  
 Created a patch that will remove the Missing Patient Register package
 and all components from the site. Posted a message in FORUM to groups 
 PROGRAMMER, CHIEF PROJECT MANAGER and SITEMANAGERS NATIONAL indicating
 the intention to remove MPR. A date of 9/30/03 was given to respond 
 with objections. No objections were received.
  
 The patch will remove the following:
  
 Print Template: MPRP LOCAL    FILE #850
  
 Sort Template: MPRS LOCAL    FILE #850
  
 Mail Groups: MPR COORDINATORS, MPR MESSAGES
  
 Bulletins: MPR Missing Patient
   
 Routines: MPRCHK, MPREFN, MPREFTSK, MPREM1, MPREM2, MPREMTSK, MPRIENV, 
 MPRIN001,MPRIN002, MPRIN003, MPRIN004,MPRIN005, MPRIN006, MPRIN007, 
 MPRIN008, MPRIN009, MPRINIS, MPRINIT, MPRINIT1, MPRINIT2, MPRINIT3, 
 MPRINIT4, MPRINIT5, MPRIPOS1, MPRIPOS2, MPRIPOST, MPRNTEG, MPRSACK 
 MPRSDATA
  
 Options:
 MPR FOUND ENTER/EDIT, MPR MAIN, MPR MISSING ENTER/EDIT, MPR MISSING 
 PATIENT PRINT, MPRSACK, MPRSDATA
  
 The following data files and DD's will be deleted:
 ^MPR(850,0) = MISSING PATIENT REGISTER^850IP^
 ^MPR(850.1,0) = MISSING PATIENT SITE PARAMETERS^850.1P^
 ^MPR(850.9,0) = NATIONAL MISSING PATIENT REGISTER^850.9I^
  
 MPR entries will be removed from the following files:
  
 Package (9.4)
 Build (9.6)
 Installs (9.7)
 Vista Applications (1200036)

 INSTALLATION INSTRUCTIONS:
 ==========================
  ***** NOTE*****
  Before installing patch, check the following:
  In DSM, make sure Access Privileges for ^MPR are as follows:
     R    = Read access
     RW   = Read and Write access
     RWP  = Read, Write, Purge access
     NONE = No access
   
  System <RWP>      World <RWP>       Group <RWP>       UCI <RWP> 
   
  In Cache, make sure global properties for ^MPR are as follows: 
  Owner=RWD         Group=RWD         World=RWD         Network=RWD
  
 ***** NOTE*****
 The install will delete routines, data files, DD, Print templates, Sort 
 templates, bulletins, and options.  You will receive a mail message 
 stating that a job has been tasked to run in 5 minutes that will complete
 the removal. The tasked job will remove MPR from files 9.4 (Package),
 9.6 (Build), 9.7 (Install) and 1200036 (VISTA Applications). You will 
 receive another mail message when the task has completed.
  
 This patch can be loaded with users in the system.  Installation time
 will be less than 15 minutes.
  
  1.   Choose the PackMan message containing this patch and invoke the 
       INSTALL/CHECK MESSAGE PackMan option.  
  
  2.   (DSM for Open VMS sites only) Disable routine mapping on all systems
       for the routines listed in the Routine Summary.  
  
       NOTE:  If the routines included in this patch are not currently in 
       your mapped routine set, please skip this step.
  
  3.   Start up the Kernel Installation and Distribution System Menu 
       [XPD MAIN]: 
  
            Edits and Distribution ...  
            Utilities ...  
            Installation ...  
  
    Select Kernel Installation & Distribution System Option: INStallation 
                                                             ---
  
            1      Load a Distribution
            2      Verify Checksums in Transport Global
            3      Print Transport Global
            4      Compare Transport Global to Current System
            5      Backup a Transport Global
            6      Install Package(s)
                   Restart Install of Package(s)
                   Unload a Distribution
  
    Select Installation Option: 
  
  4.   From this menu, you may elect to use the following options
       (When prompted for the INSTALL NAME, enter MPR*1.0*6)
  
       a.  Backup a Transport Global - This option will create a backup
           message of any routines exported with this patch. It will not
           backup any other changes such as DD's or templates.
       b.  Compare Transport Global to Current System - This option will
           allow you to view all changes that will be made when this patch
           is installed. It compares all components of this patch
           (routines, DD's, templates, etc.).
       c.  Verify Checksums in Transport Global - This option will allow 
           you to ensure the integrity of the routines that are in the
           transport global.
  
  5.   Use the Install Package(s) option and select the package MPR*1.0*6
  
       a. When prompted "Want KIDS to Rebuild Menu Trees Upon Completion
          of Install ? YES//', answer NO.
       b. When prompted 'Want KIDS to INHIBIT LOGONs during the
          install? YES//', answer NO.
       c. When prompted 'Want to DISABLE Scheduled Options and Menu 
          Options and Protocols? YES//', answer YES.
       d. When prompted 'Enter options you wish to mark as 'Out Of 
          Order':',
          enter the following options:
  
               MPR FOUND ENTER/EDIT
  
               MPR MAIN
  
               MPR MISSING ENTER/EDIT
  
               MPR MISSING PATIENT PRINT
  
               MPRSACK
  
               MPRSDATA
  
       e. When prompted 'Enter protocols you wish to mark as 'Out Of 
          Order':',press <return>.
  
  6.   (DSM for Open VMS sites only) Optional - Include the routines
       distributed with this patch in the mapped routine set. 
       NOTE: This step is only necessary if you performed step 2 or if
       you wish to include the routines in your mapped set.
  
  7.   Remove post install routine MPRBYE after you have received the 
       following mail message:
       Background Utility for MPR removal, Task #xxxxx has completed.

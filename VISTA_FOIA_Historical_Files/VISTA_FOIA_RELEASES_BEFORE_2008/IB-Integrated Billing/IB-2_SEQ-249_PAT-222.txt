Subject: WRONG TEST DATE IN PATIENT INQUIRY

Category: 
  - Routine

Description:
============

 This patch addresses a reported problem when using the Patient Inquiry 
 option [DG PATIENT INQUIRY], to view a patient.
  
  
 This patch addresses the following New Service Request (NSR):
 -------------------------------------------------------------
 There is no NSR associated with this patch.
  
  
 This patch addresses the following NOIS message(s):
 ---------------------------------------------------
 MAC-0102-61448  Means Test Date On Patient Inquiry
  
  
 Overview of NOIS Message(s):
 ----------------------------
 MAC-0102-61448  Means Test Date On Patient Inquiry
  
 Problem:
 --------
 When using the Patient Inquiry option [DG PATIENT INQUIRY] to view a 
 patient, the Means Test date that is displayed isn't always the primary 
 test date.
  
 Resolution:
 -----------
 When using the Patient Inquiry option [DG PATIENT INQUIRY] to view a 
 patient, the heading "Test date:" has been changed to "Last test date:",
 and the primary test date and reason are pulled from the patient's entry
 in the BILLING PATIENT (#354) file.
  
  
 Test Sites:
 -----------
 Dublin GA
 St. Cloud
 Clarksburg
 Salisbury
 Bay Pines

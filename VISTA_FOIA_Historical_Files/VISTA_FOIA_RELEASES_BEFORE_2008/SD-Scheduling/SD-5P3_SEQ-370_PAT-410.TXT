Subject: No Show/No Action Taken issues on No Show Report.

Category: 
  - Routine
  - Enhancement (Mandatory)

Description:
============

 This patch is to correct a problem with the No Show Report where the No
 Action Taken appointments are included with the No Show appointments, 
 therefore creating some confusion and incorrect reporting.

 REMEDY OVERVIEW:
 ================
 HD63288 File 44 and Patients Appt file with inconsistent errors.
  
 Problem:
 --------
 The No Show Report is including the No Action Taken appointments as No 
 Show appointments.
  
 Resolution:
 -----------  
 A new prompt has been added to give the user the option to select No 
 Shows Only or Both No Shows and No Action Taken appointments. The Both
 option was kept so as not to remove any of the existing functionality.
 The new prompt will show as follows:
  
 Select Outputs Option: NO-Show Report
 NO SHOW REPORT FOR WHICH DIVISION:  ALBANY// 
 Select Clinic(s): ALL// 
  
 Select one of the following:
  
   1         No Shows ONLY
   2         Both No Shows & No Action Taken
  
 Enter response: No Shows ONLY// 
  
 You may enter only a beginning date if you would like
 to see a report of No-Shows for only one date.
  
 **** Date Range Selection ****
  
    Beginning DATE :

 INSTALLATION INSTRUCTIONS:
 ==========================
 This patch can be loaded with users on the system, but since an option
 will be disabled it is recommended that it be installed at off-peak
 hours. Installation time will be less than 1 minute.
  
 1. Choose the PackMan message containing this patch and invoke the 
 INSTALL/CHECK MESSAGE PackMan option.
  
 2. Start up the Kernel Installation and Distribution System Menu [XPD 
 MAIN]:
        Edits and Distribution ...
        Utilities ...
        Installation ...
  
    Select Kernel Installation & Distribution System Option:  Installation
                                                              ---
  
        1     Load a Distribution
        2     Verify Checksums in Transport Global
        3     Print Transport Global
        4     Compare Transport Global to Current System
        5     Backup a Transport Global
        6     Install Package(s)
              Restart Install of Package(s)
              Unload a Distribution
  
 3. From this menu, you may elect to use the following options
    (When prompted for the INSTALL NAME, enter SD*5.3*410):
  
   a. Backup a Transport Global - This option will create a backup 
      message of any routines exported with this patch. It will not backup
      any other changes such as DD's or templates.
  
   b. Compare Transport Global to Current System - This option will 
      allow you to view all changes that will be made when this patch is
      installed. It compares all components of this patch (routines, DD's,
      templates, etc..)
   
   c. Verify Checksums in Transport Global - This option will allow 
      you to ensure the integrity of the routines that are in the Transport
      global.
  
 4. Use the Install Package(s) option and select the package
    SD*5.3*410.
  
   a. When prompted 'Want KIDS to INHIBIT LOGONs during the
      install? YES//', answer NO.
  
   b. When prompted 'Want to DISABLE Scheduled Options and Menu 
      Options and Protocols? YES//', answer YES.
      
      1)  When prompted to select the options you would like to mark as 
          'Out of Order', enter the following.
  
          Outputs     SDOUTPUT              
  
      2)  When prompted to select the protocols you would like to mark as 
          'Out of Order', just press <Return>.

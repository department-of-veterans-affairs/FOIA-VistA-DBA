Subject: MODIFY 'RMPR WH DISPLAY'; RECONCILING - SHIP DATES; LOCATION CASE SENSITIVE

Category: 
  - Routine
  - Print Template

Description:
============

 THIS PATCH WILL ADDRESS THREE ISSUES:
  
 1. When using the "roll and scroll" menu option to DISPLAY PROSTHETIC 
    PO INFORMATION [RMPR WH DISPLAY] option and the DELIVER TO (#19) 
    field of the PROSTHETICS 1358 (#664) file is 'other', print 
    "Deliver To Attention" on 2nd line.
  
 2. When using the Reconcile/Close Out Purchase Card Transaction 
    [RECONCILE/CLOSE OUT PURCHASE C  RMPR4 CLOSE OUT] option the
    Shipment Date will occur for date query for each item on the Purchase
    Card.
   
 3. Prosthetics location prompt is case sensitive when using the
    Reconcile Item Balance [RMPR INV RECONCILE] option.
  
  
 ASSOCIATED E3R(s):
 ==================
 N/A
  
  
 Remedy:
 ======
 1. HD120211   Display Prosthetic PO Information [RMPR WH DISPLAY] option
               does not display the delivery location for prosthetic 1358s
               marked as Deliver To: Other.
  
 2. HD117099   Shipment Date - prompted multiple times during
               PC Reconciliation.
  
 3. HD111157   Prosthetics Location is case sensitive when using the
               Reconcile Item Balance [RMPR INV RECONCILE] option.

 Remedy Overview:
 ================ 
 1. HD120211   Display Prosthetic PO Information [RMPR WH DISPLAY] option
               does not display the correct delivery location.
  
    Problem:
    --------
    Print Template Display Prosthetic PO Information [RMPR WH DISPLAY]
    option does not display the correct delivery location for other Deliver
    To which is stored in DELIVER TO ATTENTION (#21.1) field in the
    PROSTHETICS 1358 (#664) file.
  
    Resolution:
    -----------
    Modify the "roll and scroll" Print Template Display Prosthetic PO
    Information [RMPR WH DISPLAY] option to print the DELIVER TO ATTENTION
    (#21.1) field on second line of display.
  
  
 2. HD117099   Shipment Date - prompted multiple times during
               PC Reconciliation. 
  
    Problem:
    --------
    The Shipment Date is prompted for each item contained within the 
    Purchase Card.
  
    Resolution:
    -----------
    Modify RMPR4E21 and RMPR4E23 to use the default date (today or
    transaction date) without querying user for each item.
  
 3. HD111157   Prosthetics Location is case sensitive when using
               Reconcile Item Balance [RMPR INV RECONCILE] option.
  
    Problem:
    --------
    Prosthetics location is case sensitive when using the Reconcile 
    Item Balance [RMPR INV RECONCILE] option.
  
    Resolution:
    -----------
    Modify routine RMPRPIY7 to convert lower case location entry to
    upper case.

    Package(s)'.  Select the package 'RMPR*3.0*118 and proceed with the 
    install.

      (when prompted for INSTALL NAME, enter RMPR*3.0*118):

              you to ensure the integrity of the routines that are in the

              allow you to view all changes that will be made when this
              patch is installed.  It compares all components of this patch
              (routines, DD's, templates, etc.).

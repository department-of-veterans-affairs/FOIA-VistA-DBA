         **************************************************

         * IN ORDER TO PROPERLY RESPOND TO THE HURRICANE  *

         *           WITHIN 24 HOURS OF RELEASE!          *

  Registration Patch DG*5.3*677 is distributed with Income Verification
  Match patch IVM*2.0*113 in a single host file in order to:
  
  1.  Add an Emergency Response Indicator field .181 to the data dictionary
      of the PATIENT file (#2).
  2.  Create an API to retrieve the current indicator from a patient's file.
  3.  Add the indicator to the ZPD segment, sequence 40, to be transmitted
      to the HEC in the Z07 message.
  4.  Receive the indicator from the HEC in the ZPD segment, sequence 40,
      in the Z11 message.
  5.  Make the indicator enterable and editable through the Load/Edit
      Patient Data [DG LOAD PATIENT DATA], Preregister a Patient [DGPRE
      PRE-REGISTER OPTION], and Register a Patient [DG REGISTER PATIENT]
      options.
  6.  Make the indicator viewable through the Patient Inquiry [DG PATIENT
      INQUIRY] and View Registration Data [DG REGISTRATION VIEW] options.

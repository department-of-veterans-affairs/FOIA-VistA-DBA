Subject: RESOLVE NULL SUBSCRIPT IN PRCHAMYC

Category: 
  - Routine

Description:
============

 This patch corrects a Null Subscript error that can occur when a user 
 processes multiple Purchase Order amendments without exiting the option 
 prior to each amendment.

 REMEDY OVERVIEW:
 ===============
 HD55277    FGH-0202-31132  NULL SUBSCRIPT PCT1^PRCHAMYC   
   
 Problem:
 ------- 
 When a user processes multiple Purchase Orders without exiting the option 
 prior to each amendment, a null subscript error can occur because the 
 PRCH("AM") array is not being killed after entering each amendment. 
  
 Resolution:
 ----------
 Kill the PRCH("AM") array to clear existing data from previous amendments.

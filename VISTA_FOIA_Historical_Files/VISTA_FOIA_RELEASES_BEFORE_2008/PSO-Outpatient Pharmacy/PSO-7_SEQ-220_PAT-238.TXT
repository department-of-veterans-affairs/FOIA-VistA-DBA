Subject: NON-VA MEDS ISSUE/REINSTATE RX DOSAGE ISSUE

Category: 
  - Routine

Description:
============

 This patch will correct 2 problems.
  
 Problem 1:  
 If a patient has Non-VA Meds, an <UNDEFINED> error occurs when
 discontinuing all prescriptions for that patient.
  
 Problem 2:
 If a prescription is reinstating, then a different prescription is 
 selected and dosage edited, the dosages from the drug on the reinstated
 prescription display instead of the dosages from the drug on the 
 prescription being edited. Also, after editing the dosage and accepting
 the changes, the drug name is changed to the drug from the reinstated
 prescription.

 REMEDY/NOIS OVERVIEW:
 =====================
 Problem:   HD113921
 --------   HD127161
 When discontinuing all prescriptions for a patient, an error occurs when 
 processing Non-VA Meds.
  
 Resolution:
 ----------- 
 Need to modify routine PSOCAN1 at tag NAM so the Non-VA Meds are discontinued
 in the Pharmacy Patient file (#55).
  
 Problem:   HD85729
 --------   HD139338
 The drug dosages are incorrect when editing a prescription after reinstating
 a different prescription.  Also, when accepting the changes for the edited
 prescription, the drug name is changed to the drug from the reinstated 
 prescription.
  
 Resolution:
 -----------
 Need to modify routine PSOORNE2 to reset the PSODRUG array using the drug from
 the edited prescription.

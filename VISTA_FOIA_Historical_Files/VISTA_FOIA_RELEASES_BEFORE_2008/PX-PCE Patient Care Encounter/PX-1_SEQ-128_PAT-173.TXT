Subject: UNABLE TO COMPLETE CHECK OUT

Category: 
  - Routine

Description:
============

 This patch has been created to correct a problem with the 
 IHS(Indian Health Services)/PATIENT (#9000001) file and the PATIENT (#2)
 file not being in synch. This causes problems when creating the visit 
 during the Check Out process. If after installing this patch you 
 come across encounters that still can not be checked out please use the
 utility developed in PX*1.0*153. This utility was designed to correct the
 data issue brought about by the source of the IHS/PATIENT file problems.

 REMEDY/NOIS OVERVIEW:
 ==============
 1. AMA-1202-71687 (HD63386) Unable to complete Check Out.
  
 Problem:
 --------
 With the problem caused by the two patient files being out of synch, the 
 visit does not get created during the Check Out process. This causes the 
 check out process to not complete properly and thus not allow the users 
 to check out the encounter/appointment.
   
 Resolution:
 -----------
 Modifications will be made to the PXBAPI routine to check the IHS/Patient 
 file and, if needed, re-build the index to allow the check out process to
 work properly.
  
 TECHNICAL:
 ==========
 1. Problem:
    --------
 Problems with the IHS/Patient file causes problems during the check out 
 process.
  
 2. Resolution:
    -----------
 Add code to check for existing problem and correct file structure, if 
 needed.
